Kita akan melanjutkan pengembangan proyek Oklix.

Repository GitHub:

https://github.com/imamarifuddin/Oklix

Sebelum memberikan solusi:

1. Gunakan repository GitHub sebagai sumber utama.
2. Gunakan folder docs sebagai dokumentasi utama.
3. Jangan membuat asumsi berdasarkan chat lama.
4. Fokus hanya pada modul yang sedang dikerjakan.
5. Berikan file secara utuh, bukan potongan kode.
6. Jangan memperbaiki banyak modul sekaligus.
7. Setelah satu modul selesai, jalankan seluruh test modul tersebut hingga PASS.
8. Setelah PASS, baru lanjut ke modul berikutnya.

Workflow:

Repository
↓

Baca docs
↓

Analisis modul
↓

Refactor
↓

Pytest
↓

Commit
↓

Push
